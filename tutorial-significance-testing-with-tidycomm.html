7 Tutorial: Significance testing with tidycomm

After working through Tutorial 7, you’ll…

  • know how to install a package from GitHub instead of CRAN
  • know how to use the tidycomm package to run significance tests
  • get a first glimpse into quick visualizations of your significance tests using tidycomm

7.1 Installing a package from GitHub

In this part of the tutorial, we’re going to be learning how to install R packages directly from GitHub, specifically from a development branch named devel (short for: development).

Why might we want to do this? There are several reasons. Often, the most recent and innovative features of a package are found in these development branches. Developers use these branches to try out new features and fix bugs before merging them into the main branch that will be deployed via CRAN. By installing from the devel branch, you can access the latest features and updates ahead of their official release. It’s like a sneak peek into the future of the package!

However, it’s important to note that because these branches are under development, they might be unstable or have unaddressed issues. Hence, these versions are recommended for users who want to try the latest features or for those who are involved in the development of the package.

Now, let’s dive in and learn how to install the tidycomm package from the devel branch on GitHub!

First, you will need to install and load the devtools package, a package that support R package development. In addition, install and load the `remotes`` package to install from Github.

# install.packages("devtools") run only the first time
# install.packages("remotes") run only the first time
library(devtools)
library(remotes)

Next, we’ll need to check our internet browser and navigate to Julian Unkel’s Github repository that stores tidycomm. Here you can see that Julian has several branches in this repository, one of which is called devel.

Let’s install the tidycomm package from this devel branch by using the install.github function and providing it with the GitHub user name, the repository name and the branch_name:

remotes::install_github("joon-e/tidycomm@devel")

After that, just load the package:

library(tidycomm)

And get the WoJ data:

WoJ <- tidycomm::WoJ

7.2 How to run significance tests with tidycomm

7.2.1 t_test()

The t_test() function is used to compute t-tests for one group variable and specified test variables. If no variables are specified, all numeric (integer or double) variables are used.

You can provide an independent (group) variable and a dependent variable:

WoJ %>% t_test(temp_contract, autonomy_selection)
## # A tibble: 1 × 10
##   Variable M_Permanent SD_Permanent M_Temporary SD_Temporary Delta_M     t    df
## * <chr>          <dbl>        <dbl>       <dbl>        <dbl>   <dbl> <dbl> <dbl>
## 1 autonom…        3.91        0.755        3.70        0.932   0.212  1.96   998
## # ℹ 2 more variables: p <dbl>, d <dbl>

If you specify no dependent variable, all suitable variables in your data set will be used:

WoJ %>% t_test(temp_contract)
## # A tibble: 11 × 10
##    Variable     M_Permanent SD_Permanent M_Temporary SD_Temporary Delta_M      t
##  * <chr>              <dbl>        <dbl>       <dbl>        <dbl>   <dbl>  <dbl>
##  1 autonomy_se…        3.91        0.755        3.70        0.932  0.212   1.96 
##  2 autonomy_em…        4.12        0.768        3.89        0.870  0.237   2.17 
##  3 ethics_1            1.57        0.850        1.98        0.990 -0.414  -3.41 
##  4 ethics_2            3.24        1.26         3.51        1.23  -0.269  -1.51 
##  5 ethics_3            2.37        1.12         2.28        0.928  0.0862  0.549
##  6 ethics_4            2.53        1.24         2.57        1.22  -0.0323 -0.185
##  7 work_experi…       17.7        10.5         11.3        11.8    6.42    4.29 
##  8 trust_parli…        3.07        0.797        3.02        0.772  0.0539  0.480
##  9 trust_gover…        2.87        0.847        2.64        0.811  0.229   1.92 
## 10 trust_parti…        2.43        0.724        2.36        0.736  0.0719  0.703
## 11 trust_polit…        2.53        0.707        2.40        0.689  0.136   1.37 
## # ℹ 3 more variables: df <dbl>, p <dbl>, d <dbl>

If you have an independent (group) variable with more than two levels, you can also specifiy the levels that you want to use for your t_test():

WoJ %>% t_test(employment, autonomy_selection, levels = c("Full-time", "Freelancer"))
## # A tibble: 1 × 10
##   Variable `M_Full-time` `SD_Full-time` M_Freelancer SD_Freelancer Delta_M     t
## * <chr>            <dbl>          <dbl>        <dbl>         <dbl>   <dbl> <dbl>
## 1 autonom…          3.90          0.782         3.76         0.993   0.139  2.03
## # ℹ 3 more variables: df <dbl>, p <dbl>, d <dbl>

Finally, t_test() comes with additional arguments. For example, the var.equal parameter indicates whether to treat the two variances as being equal. If TRUE then the pooled variance is used to estimate the variance otherwise the Welch (or Satterthwaite) approximation to the degrees of freedom is used. It defaults to TRUE.

WoJ %>% t_test(temp_contract, autonomy_selection, var.equal = TRUE)
## # A tibble: 1 × 10
##   Variable M_Permanent SD_Permanent M_Temporary SD_Temporary Delta_M     t    df
## * <chr>          <dbl>        <dbl>       <dbl>        <dbl>   <dbl> <dbl> <dbl>
## 1 autonom…        3.91        0.755        3.70        0.932   0.212  1.96   998
## # ℹ 2 more variables: p <dbl>, d <dbl>

vs.

WoJ %>% t_test(temp_contract, autonomy_selection, var.equal = FALSE)
## # A tibble: 1 × 10
##   Variable M_Permanent SD_Permanent M_Temporary SD_Temporary Delta_M     t    df
## * <chr>          <dbl>        <dbl>       <dbl>        <dbl>   <dbl> <dbl> <dbl>
## 1 autonom…        3.91        0.755        3.70        0.932   0.212  1.63  55.9
## # ℹ 2 more variables: p <dbl>, d <dbl>

You can also run a one-sample-ttest or location test by providing a variable and checking whether the mean of this variable is equal to a provided population mean mu:

WoJ %>% t_test(autonomy_selection, mu = 3)
## # A tibble: 1 × 9
##   Variable               M    SD CI_95_LL CI_95_UL    Mu     t    df         p
## * <chr>              <dbl> <dbl>    <dbl>    <dbl> <dbl> <dbl> <dbl>     <dbl>
## 1 autonomy_selection  3.88 0.803     3.83     3.92     3  37.7  1196 6.32e-206

Please remember that you can access the full documentation of this function using:

?t_test()

7.2.2 unianova()

The unianova() function is used to compute one-way ANOVAs for one group variable and specified test variables. If no variables are specified, all numeric (integer or double) variables are used.

unianova works pretty similar to t_test():

WoJ %>% unianova(employment, autonomy_selection)
## # A tibble: 1 × 6
##   Var                    F df_num df_denom      p eta_squared
## * <chr>              <dbl>  <dbl>    <dbl>  <dbl>       <dbl>
## 1 autonomy_selection  2.42      2     1194 0.0896     0.00403

Provide only an independent variable to use all suitable avriables in your data set as dependent variables:

WoJ %>% unianova(employment)
## # A tibble: 11 × 6
##    Var                     F df_num df_denom          p eta_squared
##  * <chr>               <dbl>  <dbl>    <dbl>      <dbl>       <dbl>
##  1 autonomy_selection  2.42       2     1194 0.0896        0.00403 
##  2 autonomy_emphasis   5.86       2     1192 0.00293       0.00974 
##  3 ethics_1            2.17       2     1197 0.115         0.00361 
##  4 ethics_2            2.20       2     1197 0.111         0.00367 
##  5 ethics_3            5.33       2     1197 0.00498       0.00882 
##  6 ethics_4            3.45       2     1197 0.0320        0.00574

7.2.3 correlate()

The correlate()function computes correlation coefficients for all combinations of the specified variables. If no variables are specified, all numeric (integer or double) variables are used.

WoJ %>% correlate(ethics_1, autonomy_selection)
## # A tibble: 1 × 5
##   x        y                        r    df       p
## * <chr>    <chr>                <dbl> <int>   <dbl>
## 1 ethics_1 autonomy_selection -0.0766  1195 0.00798

Specify more than two variables:

WoJ %>% correlate(ethics_1, autonomy_selection, work_experience)
## # A tibble: 3 × 5
##   x                  y                        r    df            p
## * <chr>              <chr>                <dbl> <int>        <dbl>
## 1 ethics_1           autonomy_selection -0.0766  1195 0.00798     
## 2 ethics_1           work_experience    -0.103   1185 0.000387    
## 3 autonomy_selection work_experience     0.161   1182 0.0000000271

Or calculate the partial correlation for three specified variables:

WoJ %>% correlate(ethics_1, autonomy_selection, work_experience, partial = TRUE)
## # A tibble: 3 × 6
##   x               y                  z                        r    df          p
## * <chr>           <chr>              <chr>                <dbl> <dbl>      <dbl>

## 2 ethics_1        work_experience    autonomy_selection -0.0924  1181    1.46e-3

Get the correlation for all suitable variables in your data set:

7.2.4 regress()

Of course, you can also run linear regressions with the regress() function:

WoJ %>% regress(ethics_1, autonomy_selection)
## # A tibble: 2 × 6
##   Variable                 B StdErr    beta     t        p
## * <chr>                <dbl>  <dbl>   <dbl> <dbl>    <dbl>
## 1 (Intercept)         1.96   0.127  NA      15.4  4.57e-49
## 2 autonomy_selection -0.0852 0.0321 -0.0766 -2.66 7.98e- 3
## # F(1, 1195) = 7.061023, p = 0.007983, R-square = 0.005874

Use two dependent variables:

WoJ %>% regress(ethics_1, autonomy_selection, work_experience)
## # A tibble: 3 × 6
##   Variable                  B  StdErr    beta     t        p
## * <chr>                 <dbl>   <dbl>   <dbl> <dbl>    <dbl>
## 1 (Intercept)         2.03    0.129   NA      15.8  5.32e-51
## 2 autonomy_selection -0.0692  0.0325  -0.0624 -2.13 3.32e- 2

## # F(2, 1181) = 8.677001, p = 0.000182, R-square = 0.014482

Again, you can always get help:

?regress()
